    Checked vs unchecked #

    Unchecked exceptions #

    The simplest type of exception in Java is called an unchecked exception. An unchecked exception is an instances of the native class RuntimeException.

    The class RuntimeException has several native subclasses. You may be familiar wit some of them: NullPointerException, ArrayIndexOutOfBoundsException, IllegalArgumentException, etc.

    An unchecked exception that is not caught is automatically rethrown.

    Checked exceptions #

    In Java, an instance of Exception that is not an instance of RuntimeException is a checked exception.

    For instance, instances of the class FileNotFoundException are checked exceptions.

    A checked exception is not rethrown by default. Instead, it must be explicitly rethrown (or caught), otherwise the program does not compile.

    For instance, the following code does not compile, because the constructor of FileInputStream may throw a FileNotFoundException.

    void myMethod(String path) {
    FileInputStream stream = new FileInputStream(path);
        ...
}

    This can be fixed by either catching the exception, or explicitly rethrowing it, as follows:

    void myMethod(String path) throws FileInputStream {
    FileInputStream stream = new FileInputStream(path);
        ...
}

    Then a method that calls myMethod will in turn have to catch the exception, or rethrow it explicitly, etc.

    Custom exceptions #

    You can create your own exception types by extending one of Java’s native exception types. For instance:

    public class WrongNumberOfRowsException extends IllegalArgumentException {

    public WrongNumberOfRowsException(int numberOfRows) {
        super("This sudoku grid has "+numberOfRows+" rows, whereas it should have 9").
    }
}

    A custom exception can be thrown like a regular one:

    if(sudokuGrid.length != 9){
    throw new WrongNumberOfRowsException(sudokuGrid.length);
}

    Warning. Recall that an instance of a subclass of RuntimeException is unchecked.

    In this example, WrongNumberOfRowsException extends IllegalArgumentException, which extends RuntimeException. Therefore an instance of WrongNumberOfRowsException is an unchecked exception.

    Warning. Checked exceptions add boilerplate code to signature methods (e.g. throws FileInputStream in the above example). For this reason, it is often recommended to use unchecked exceptions (i.e. instances of RuntimeException) by default. In Effective java (Item 71) Joshua Bloch suggests unchecked exceptions only if some calling method is likely to catch the exception and take a meaningful action when this happens.
