    A simplified model #

    As we saw earlier, parallelism and concurrency are two different execution strategy. However, (as we discussed) in a high-level programming language like Java, if a program allows two threads to be executed concurrently (on the same core), then it also allows them to be executed in parallel (on different cores).

    From an abstract perspective, in order to understand the properties of a such program it can be convenient to assume an simplified, ideal machine that:

    • has an unlimited number of cores, and
    • relies exclusively on parallelism.

    Observation. Even if two thread are executed in parallel (i.e by different cores), they cannot access the same resource (value, object, array) at the same time.

    Parallel computation #

    In this section, we borrow (and further simplify) the simple syntax for parallel computation used in the popular textbook Introduction to algorithms.

    It extends imperative pseudocode with the keywords “spawn”, “sync”, and “parallel”. These keywords correspond to primitives available (in one form or another) in many programming languages (like Java) or libraries (like OpenMP) that support multithreading.

    spawn and sync #

    In this model:

    • the keyword spawn allows a method to create a child thread and execute an instruction in this thread. In pseudocode, this is written $\qquad$ spawn <myInstruction>

    • the keyword sync force the current method to interrupt its execution, until all the threads that it has spawn have terminated. In pseudocode, this is written as a single “instruction” $\qquad$ sync

    Example.

    The following method:

    • takes two vectors of integers as input,
    • computes the sum of the elements of each vector, and returns the sum of the two results.
    int aggregate (int[] v1, int[]v2){

    int s1 = spawn sumElts(v1)

    int s2 = sumElts(v2)

    print("waiting")
    sync

    return s1 + s2
}

int sumElts(int[] v){
    int sum = 0
    for(int i = 0; i v.length; i++){
        sum += v[i]
    }
    print("done")
    return sum
}

    In this method:

    • the “instruction” spawn sumElts(v1) creates a child thread and executes a call to sumElts in this thread.
    • the “instruction” sync forces the main thread to wait for completion of the child thread.

    Observe that there is no guarantee on which thread will first finish executing the method sumElts. As a result, depending on the runtime environment, the program may output either

    done
waiting
done

    or

    done
done
waiting

    parallel #

    The keyword parallel is associated to a loop, and intuitively allows the execution environment to execute iterations of this loop in different threads (for instance, the library OpenMP provides such a keyword).

    The (abstract) syntax is the following:

    for parallel (<myLoopSpecification>){
    <myLoopBody>
}

    Example. We can parallelize the method sumElts in our previous example by simply adding the keyword parallel to the loop:

    int sumElts(int[] v){
    int sum = 0
    for parallel (int i = 0; i v.length; i++){
        sum += v[i]
    }
    return sum
}

    The keyword can always be replaced with combinations of spawn and (if needed) sync.

    Example. The above example can be rewritten as follows

    int sumElts(int[] v){
    sumRec(v, 0, v.length - 1)
}

int sumRec(int[] v, int i, int max){
    // base case
    if(i = max){
        return v[i]
    }
    // inductive case
    // compute the middle index
    int mid = (i + max) / 2
    // recursive call on the left half, in a separate thread
    int leftSum = spawn sumRec(v, i, mid)
    // recursive call on the right half (in the current thread)
    int rightSum = sumRec(v, mid + 1, max)

    sync
    return leftSum + rightSum
}

    Definition. The span of a program’s execution is the maximal number of instructions it has to perform sequentially.

    Exercise: Let $f(n)$ measure the span of the program above, where $n$ d=is the size of the input vector. Assuming a number of cores $\ge n$, which asymptotic class of cost functions does $f$ belong to?

    Solution: $f$ is in $Theta(log n)$

    Note. In this example, it may be tempting to spawn the threads sequentially, rather than recursively. However, the benefit of parallelization is lost:

    int sumElts(int[] v){
    int sum = 0
    for (int i = 0; i v.length; i++){
        spawn sum += v[i]
        sync
    }
    return sum
}

    in Java #

    The main method of a Java program is executed in a thread called the main thread.

    Creating a thread #

    A Java thread is an instance of the class Thread.

    There are many alternative syntaxes to create an instance of this class.

    We cover here two of the most common ones.

    instantiate Runnable #

    The constructor Thread(Runnable runnable) takes ans input an instance of the interface Runnable.

    Runnable is a function interface (whose method is called run).

    Therefore is can be instantiated with a lambda expression.

    Example.

    Thread myThread = new Thread(() -> {System.out.println("Hello World");});

    Alternatively, in order to pass an argument to the thread, we can create a class that implemented Runnable, and pass the argument to the constructor if this class.

    public class MyRunnable implements Runnable {

    int arg;

    public MyRunnable(int arg){
        this.arg = arg;
    }
:
    public void run() {
        System.out.println(arg);
    }
}

    Thread myThread = new Thread(new MyRunnable(4));

    create a subclass of Thread #

    Spawning a thread #

    The instance method Thread.start spawns a thread.

    Example.

    Thread myThread = new Thread(() -> {System.out.println("Hello World");});
myThread.start();

    Waiting for a thread to terminate #

    Java provides a functionality analogous to the sync keyword seen above, but with a more fine-grained control.

    The instance method Thread.join called on a thread $t$ causes the current thread to pause its execution until termination of $t$.

    Thread myThread = new Thread(() -> { myExpensiveMethod(); });
myThread.start();

// wait for the spawned thread to terminate
myThread.join()

System.out.println("Job done, resuming execution");

    Loop #

    In Java, an analogous of the keyword sleep is the instance method Stream.parallel that we saw in the dedicated chapter.

    Example.

    List<Integer> vector = getVector();

vector.stream
    .parallel
    .filter(<myPredicate>)
    .map(<myFunction>)
    .forEach(<myConsumer>);

    In this case, the whole stream pipeline is executed in a concurrent fashion (or parallel if some cores are available).

    By default, a pool of threads of size $k - 1$ is used, where $k$ is the number of cores of the CPU (but a larger pool can be manually configured).

    Pause #

    The static method Thread.sleep can be used to pause the current thread for a given amount of time.

    Example.

    System.out.ptinln("Going to sleep...");

// sleep 4 seconds
Thread.sleep(4000);

System.out.ptinln("Waking up");

    Warning. The method Thread.sleep does not guarantee that the input duration is precisely enforced. Therefore sleeping times should not be used to synchronize the activity of two threads.
